
- Profile updates

## 🔌 Socket.IO Module

### Main File:
- `src/socket/index.ts`

### Auth Middleware:
- `src/socket/middleware/auth.ts` - JWT verification

### Event Handlers:
- `src/socket/handlers/message.handler.ts` - Message send & seen
- `src/socket/handlers/typing.handler.ts` - Typing indicators

### Features:
- JWT authentication for connections
- In-memory online users map
- Online/offline event broadcasting
- Message delivery via Socket.IO
- Typing indicators
- Error handling

### Socket Events:

**Client → Server:**
- `message:send` - Send message with conversationId & text
- `message:seen` - Mark message as seen
- `typing:start` - Start typing indicator
- `typing:stop` - Stop typing indicator

**Server → Client:**
- `message:new` - New message received
- `message:seen:update` - Message marked as seen
- `typing:update` - Typing status changed
- `user:online` - User came online
- `user:offline` - User went offline
- `error` - Error occurred

## 🗄️ Database Models

### User Model (`src/models/user.model.ts`)
```javascript
{
  username: String (unique, 3+ chars, lowercase),
  name: String (required),
  email: String (unique, valid format),
  password: String (hashed with bcrypt, select: false),
  avatar: String (optional),
  bio: String (max 500 chars),
  location: GeoJSON Point {[lng, lat]} (for geospatial queries),
  lastActive: Date,
  createdAt: Date
}
Indexes: username, location (2dsphere)
```

### FriendRequest Model (`src/models/friend-request.model.ts`)
```javascript
{
  sender: ObjectId (ref User),
  receiver: ObjectId (ref User),
  status: String (pending|accepted|rejected),
  createdAt: Date
}
Indexes: sender+receiver+status, receiver+status
```

### Conversation Model (`src/models/conversation.model.ts`)
```javascript
{
  members: [ObjectId] (ref User, supports group chats),
  lastMessage: String (cached for UI),
  lastMessageAt: Date (for sorting),
  createdAt: Date
}
Indexes: members
```

### Message Model (`src/models/message.model.ts`)
```javascript
{
  conversationId: ObjectId (ref Conversation),
  sender: ObjectId (ref User),
  text: String (required, trimmed),
  seenBy: [ObjectId] (ref User, who viewed it),
  createdAt: Date
}
Indexes: conversationId+createdAt, sender
```
